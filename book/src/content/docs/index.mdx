## Who Is This For

This material is for security professionals conducting **authorized** testing of Android identity verification systems. Whether you're a penetration tester adding biometric bypass to your toolkit, a security engineer validating your company's KYC flow, or a developer who wants to understand how these attacks work so you can defend against them — this book gives you the methodology, tools, and practice targets.

**Prerequisites:** Comfort with the command line, basic Android development concepts, and a willingness to read bytecode. No prior reverse engineering experience required — Part I covers the foundations.

## The Methodology

Every engagement follows the same cycle:

1. **Recon** — Decode the APK, map every hookable surface
2. **Patch** — Instrument the bytecode with the patch-tool
3. **Configure** — Push payloads (frames, GPS configs, sensor profiles)
4. **Execute** — Run the target flow with all injection subsystems active
5. **Report** — Capture evidence, compute statistics, write findings

You'll learn each phase independently, then combine them in full engagements against multi-step verification flows.
